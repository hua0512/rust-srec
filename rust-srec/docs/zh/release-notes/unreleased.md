# 更新日志

## `unreleased`

### 修复

- **管道执行详情在移动端正常显示**

  在窄屏上，管道执行详情页的概览卡片可能出现图标重叠、进度百分比等较长数值被截断，以及卡片图标高低不齐的情况。现在这些卡片会自适应小屏幕，图标也会对齐排列，进度、步骤数和开始时间都能正常显示。

- **主题更改重新即时生效**

  此前如果你已经自定义过主题，在主题页面选择新主题后需要刷新页面才能生效。现在主题预设、颜色和圆角的更改都会立即生效。

- **列表的筛选和搜索在返回时会保留**

  在主播、会话、管道任务、预设、工作流、媒体产物等页面上，此前只要打开某一项再返回、或刷新页面，你的搜索内容、筛选、排序和所在页码都会被重置。现在这些状态会保存在页面地址中，返回或刷新都会停留在原处，你还可以收藏或分享某个筛选后的视图。

- **分页按钮现在会跟随界面语言**

  在分页列表中，“上一页”和“下一页”按钮此前即使界面切换到其他语言仍显示英文。现在它们会以你选择的语言显示。

- **需要登录的直播流在内置播放器中稳定播放**

  对于播放列表需要 Cookie 或自定义请求头的直播流，此前只有第一个请求会带上这些信息，导致后续在网页端和桌面端播放器中卡住或失败。现在播放器会在整个播放列表中都带上这些请求头——包括清晰度变体、分片、加密密钥和低延迟分段——让这些直播流稳定播放。

- **新增用于本地网络流来源的设置**

  为避免内置播放器的代理被指向私有地址，现在它默认只访问公网上的流来源。如果你要观看或录制来自本地网络的来源——例如局域网转推、摄像头或 tailnet 上的设备——请在“网络与系统”设置中开启 **允许私有网络流代理目标** 以允许访问。

- **中断的 rclone 移动上传现在可以在重试后完成**

  当移动上传已发送部分文件后中途失败（例如网络波动）时，此前每次重试都会继续失败，因为已上传的文件在本地已不存在，手动重试任务也会遇到同样的错误。现在重试会从上次中断的地方继续，只上传剩余的文件；如果所有文件都已在之前的尝试中上传完成，重试的任务会直接成功。

- **中断的本地文件移动同样可以在重试后恢复**

  复制/移动管道步骤也存在同样的问题：如果移动在部分文件已到达目标文件夹后被中断，重试任务会把这些文件报告为失败；如果所有文件都已移动完成，整个任务会直接失败。现在重试会识别已到达目标位置的文件并正常完成。跨盘移动也会先复制到临时文件名，因此中断的移动不会再在最终文件名下留下写了一半的文件。

- **取消或超时的任务不再让工具在后台继续运行**

  此前当管道任务被取消或超过任务超时时间时，它启动的外部工具——rclone 传输、ffmpeg 处理（转封装、转码、字幕压制、缩略图）、弹幕转换或 Telegram 下载——即使任务已被标记为失败，仍可能在后台继续运行；rclone 的临时文件列表也会残留在录制文件夹中。现在停止任务会同时停止这些工具并清理临时文件。
