# 更新日志

## `unreleased`

本次更新继续完善 `v0.3.0` 中的录制会话稳定性改进，修复了几类
rust-srec 可能过早结束会话，或让仪表盘/时间线与实际录制状态略有
不同步的情况。

## 录制稳定性

- **临时下载失败更不容易误结束会话**

  某些下载失败此前可能被当成更强的"直播已离线"信号。rust-srec
  现在会更准确地区分是哪种下载引擎、哪种流类型报告了失败，因此临时
  网络问题更不容易把一次录制切成多个会话。

- **录制排程关闭时，会话结束更干净**

  当直播离开允许录制的时间段时，rust-srec 现在会通过正常的会话流程
  关闭当前录制。这能让仪表盘、通知、后处理以及会话时间线都与排程
  决定保持一致。

- **重复的排程外检查更安静**

  如果主播已经处于排程外，且没有正在录制的会话，后续重复检查不会再
  持续在后台写入相同的会话状态。

- **会话时间线数据更一致**

  会话时间线事件现在会与会话详情页的其他数据走同一套处理方式。页面
  展示不会变化，但当存在旧的或格式异常的事件详情时，时间线会更可靠。
